Here's a complete map of the project and where to edit for common changes:
---
Root Level
Security Group Webpage/
├── requirements.txt          → Python dependencies (Django, Pillow, etc.)
├── README.md                 → Project documentation & run instructions
├── venv/                     → Virtual environment (DO NOT EDIT)
└── securitywebpage/          → The actual Django project
---
securitywebpage/ — Django Project
File/Folder	Purpose
manage.py	Command-line tool for running the project (runserver, migrate, etc.)
db.sqlite3	SQLite database file — stores all your data
media/	Uploaded images (member photos, research topic images)
static/	CSS, JS, images served to the browser
templates/	HTML files the user sees
insait_security_group_webpage/	Project settings — global configuration
core/	Main app — models, views, admin, URLs
---
Where to Edit — By Task
Change site-wide settings (installed apps, database, static/media)
→ insait_security_group_webpage/settings.py
Change URL routes
→ insait_security_group_webpage/urls.py (project-level, includes core URLs)
→ core/urls.py (app-level, maps paths to views)
Change the database structure (add/remove fields)
→ core/models.py — then run:
python manage.py makemigrations
python manage.py migrate
Change what data is loaded into pages
→ core/views.py — controls which models/objects are passed to templates
Change how the admin panel looks or which fields show
→ core/admin.py — register models, customize list displays, filters, fieldsets
Change the HTML/layout of a page
→ templates/base.html — navbar, footer, shared structure (all pages extend this)
→ templates/core/home.html — Home page
→ templates/core/research.html — Research page
→ templates/core/members.html — Members page
→ templates/core/publications.html — Publications page
→ templates/core/contact.html — Contact page
→ templates/admin/base_site.html — Admin panel styling
Change styling (colors, fonts, animations, spacing)
→ static/css/style.css — All custom CSS lives here
Change JavaScript behavior (scroll effects, animations)
→ static/js/main.js
Change sample/seed data
→ core/management/commands/seed_data.py
